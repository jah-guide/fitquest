# Submission Evidence Checklist

Add the following evidence files before final hand-in:

- [ ] Demo recorded on a physical phone (not emulator) with voice-over
- [ ] Demo video link added in root `README.md`
- [ ] Video shows registration and login flow (password auth)
- [ ] Video shows SSO flow (Google and/or Apple)
- [ ] Video shows settings changes (theme/language/notifications)
- [ ] Video shows REST API + database-backed features (e.g. routines/profile)
- [ ] Video shows offline usage and later sync behavior
- [ ] Video shows what data is stored in API/database
- [ ] Signed APK evidence screenshot/export
- [ ] Feature screenshots (home, auth, settings, routines, offline/sync)
- [ ] Google Play Console upload/publish screenshot (if applicable)
- [ ] API/database evidence screenshot from hosted service
- [ ] Root `README.md` present and fully updated
- [ ] `RELEASE_NOTES.md` lists prototype -> final changes and innovations
- [ ] GitHub Actions checks passing on latest `master`/`main` commit
- [ ] Final POE commit tagged (e.g. `Final-POE`)

Suggested structure:

- `submission-evidence/screenshots/`
- `submission-evidence/video/`
- `submission-evidence/release/`

Reference guides:

- `submission-evidence/README.md`
- `submission-evidence/screenshots/README.md`
